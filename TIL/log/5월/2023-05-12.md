## API의 단일책임?

뭔가 API URL이 단일책임을 가진다는 시각과, API 사용자가 사용하기 편하도록 다양한 행위가 조합된다는 시각의 충돌이 있었다.

DAO와 Repository의 관점과 유사한 개념이다.

결론은 프론트서버라는게 있고, 일할 때 단일개념을 개발한걸 프론트서버에서 조합하고 클라이언트가 사용하기 용이하게 만들어준다고 한다.
